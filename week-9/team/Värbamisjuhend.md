**Meeskond**: Tooteanalüüsi meeskond

## 1. ÜLEVAADE

See peatükk käsitleb, kuidas hinnata Junior Data Analyst kandidaadi tehnilisi oskusi UrbanStyle’i värbamisprotsessis. Fookuses on SQL, Python, visualiseerimine, GitHubi kasutamine ja kandidaatide võime siduda tehniline töö äriliste eesmärkidega. Tugev tehniline analüütik ei kirjuta ainult koodi, vaid suudab ka selgitada, kuidas tema töö aitab ettevõttel teha paremaid otsuseid.

## 2. HINDAMISKRITEERIUMID

- Kas projektidel on mõõdetav äriline mõju?

- Kas README failid annavad kiire ülevaate projekti eesmärgist ja tulemusest?
- Kas projekt sobiks LinkedIni või CV tugevaks näiteks?

## 3. KONKREETSED NÄITED (UrbanStyle kontekst)
- Kandidaat kasutas SQL-is CTE-sid ja `ROW_NUMBER()` funktsioone kliendikäitumise analüüsimiseks, mis näitas tugevat tehnilist taset.
- Pythoni pipeline jagamine mitmeks failiks (`data_fetcher.py`, `transform.py`, `visualize_export.py`) näitas arusaamist tootmistaseme arhitektuurist.
- Kandidaat märkas puuduvat `stores` tabelit ja leidis alternatiivse lahenduse olemasolevate andmete põhjal — see näitas analüütilist kohanemisvõimet.
- Power BI dashboardides kasutati KPI-kaarte ja ärilisi annotatsioone (“mai anomaalia”), mis muutsid andmed juhtkonnale arusaadavaks.
- Regulaarne GitHub commit-ajalugu ja põhjalikud README failid näitasid tugevaid tööharjumusi ja dokumenteerimisoskust.
- Tugevad projektid sisaldasid mõõdetavaid tulemusi: ROI analüüs, duplikaatide leidmine, automatiseeritud pipeline ja ärilised soovitused juhtkonnale.

## 4. 3 VÕTMESOOVITUST JUHILE
* **Soovitus 1**: Anna kandidaatidele praktiline SQL või Python ülesanne, mis põhineb päris äriprobleemil, mitte ainult teoorial.
* **Soovitus 2**: Kontrolli "Ja mis siis?" testi. Palu kandidaadil selgitada oma portfoolio kõige keerulisemat projekti ning jälgi, kas ta jõuab tehniliselt teostuselt ärilise soovituseni või jääb kinni koodi kirjeldamisse.
* **Soovitus 3**: Prioritiseeri dokumentatsioonikultuuri. Vaata GitHubi commit ajalugu ja README kvaliteeti — need näitavad tööharjumusi ja dokumenteerimisoskust. Kuna UrbanStyle kasutab ka remote-töötajaid (Soome, Saksamaa), on iseseisvus ja põhjalikud README-failid kriitilised, et tagada analüüsiprotsessi usaldusväärsus ilma pideva järelevalveta.

## 5. ÜLLATAV AVASTUS
Kõige tugevamad kandidaadid paistsid silma mitte ainult tehniliste oskustega, vaid oskusega dokumenteerida oma tööprotsessi ja selgitada keerulisi analüüse juhtkonnale arusaadavas keeles.
